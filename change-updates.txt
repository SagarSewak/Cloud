# Change Updates Log

This file documents the step-by-step implementation of bug fixes and change requests from the training backlog.

## MOD1: account-service

### MOD1-CR-01: Decouple RestTemplate Bean
- **Goal:** Move `RestTemplate` bean declaration from the main application class to a separate configuration class.
- **Steps:**
    1. Created `com.gdb.account.config.RestTemplateConfig` class.
    2. Moved `@Bean` method for `RestTemplate` from `AccountServiceApplication.java` to `RestTemplateConfig.java`.
    3. Added `@Configuration` annotation to `RestTemplateConfig.java`.

### MOD1-BUG-01: NPE on validation helper injection
- **Goal:** Register `AccountValidator` as a Spring component to prevent `NullPointerException` during injection.
- **Steps:**
    1. Added `@Component` annotation to `AccountValidator.java`. (Verified it is present)

## MOD2: transactions-service (AOP)

### MOD2-CR-01 & MOD2-BUG-01: Performance Monitoring and Double Execution Fix
- **Goal:** Monitor execution metrics and fix double execution in `LoggingAspect`.
- **Steps:**
    1. Modified `LoggingAspect.java` in `transactions-service`.
    2. Changed `@Around` advice to call `joinPoint.proceed()` exactly once.
    3. Captured the result of `proceed()` and returned it to the caller.
    4. Logged the start time, method name, and calculated duration for performance monitoring.

## MOD3: transactions-service (JDBC)

### MOD3-CR-01: Upgrade to Named Parameters
- **Goal:** Replace `JdbcTemplate` with `NamedParameterJdbcTemplate` for better readability and maintainability.
- **Steps:**
    1. Updated `FundTransferRepositoryImpl.java` to use `NamedParameterJdbcTemplate`.
    2. Rewrote all SQL queries to use named parameters (e.g., `:fromAccount`, `:toAccount`).
    3. Used `MapSqlParameterSource` to bind values to named parameters.

### MOD3-BUG-01: SQL parameters count mismatch
- **Goal:** Fix `IndexOutOfBoundsException` in `findByAccount` method.
- **Steps:**
    1. Identifed that the positional query `WHERE from_account = ? OR to_account = ?` was missing one binding argument.
    2. By switching to `NamedParameterJdbcTemplate` in MOD3-CR-01, the issue was naturally resolved as the same named parameter `:accountNumber` can be reused multiple times in the query.
